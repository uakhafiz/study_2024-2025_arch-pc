## 2. Теоретическое введение

### 2.1 Понятие подпрограммы

Подпрограмма — это функционально законченный участок кода, который можно многократно вызывать из разных мест программы. В отличие от простых переходов, из подпрограмм существует возврат на команду, следующую за вызовом.

**Ключевые инструкции:**
| Инструкция | Описание |
|------------|----------|
| `call метка` | Вызов подпрограммы (сохраняет адрес возврата в стек) |
| `ret` | Возврат из подпрограммы (извлекает адрес возврата из стека) |

### 2.2 Механизм работы подпрограмм

При вызове подпрограммы:
1. Адрес следующей инструкции сохраняется в стеке
2. Управление передается на первую инструкцию подпрограммы
3. По завершении подпрограммы адрес возврата извлекается из стека
4. Выполнение продолжается с инструкции, следующей за `call`

### 2.3 Основные возможности отладчика GDB

GDB (GNU Debugger) — мощный отладчик, предоставляющий следующие возможности:
- Установка точек останова (breakpoints)
- Пошаговое выполнение программы
- Просмотр и изменение регистров и памяти
- Дизассемблирование кода
- Анализ стека вызовов

**Основные типы точек останова:**
- **Breakpoint** - остановка при достижении определенной строки или адреса
- **Watchpoint** - остановка при изменении значения переменной

## 3. Выполнение лабораторной работы

### 3.1 Программа lab09-1.asm - Базовая работа с подпрограммами

**Задача:** Создание программы вычисления выражения `f(x) = 2x + 7` с использованием подпрограммы.

**Исходный код:**
```asm
; lab09-1.asm
; Программа вычисления f(x) = 2x + 7

%include 'in_out.asm'

SECTION .data
    msg: DB 'Введите x: ',0
    result: DB '2x+7=',0

SECTION .bss
    x: RESB 80
    res: RESB 80

SECTION .text
    GLOBAL _start

_start:
    mov eax, msg
    call sprint
    mov ecx, x
    mov edx, 80
    call sread
    mov eax, x
    call atoi
    call _calcul
    mov eax, result
    call sprint
    mov eax, [res]
    call iprintLF
    call quit

_calcul:
    mov ebx, 2
    mul ebx
    add eax, 7
    mov [res], eax
    ret
```

Результат выполнения:

```bash

2x+7=17
```

3.2 Модификация программы с вложенными подпрограммами

Задача: Модификация программы для вычисления сложной функции f(g(x)), где:

· g(x) = 3x - 1
· f(x) = 2x + 7

Модифицированный код:

```asm
; lab09-1.asm (модифицированный)
; Программа вычисления f(g(x)) = 2*(3x-1)+7

%include 'in_out.asm'

SECTION .data
    msg: DB 'Введите x: ',0
    result: DB 'f(g(x))=2*(3x-1)+7=',0

SECTION .bss
    x: RESB 80
    res: RESB 80
    temp: RESB 80

SECTION .text
    GLOBAL _start

_start:
    ; Ввод x
    mov eax, msg
    call sprint
    mov ecx, x
    mov edx, 80
    call sread
    mov eax, x
    call atoi

    ; Вычисление g(x) = 3x - 1
    call _g

    ; Сохранение результата g(x)
    mov [temp], eax

    ; Вычисление f(g(x)) = 2 * g(x) + 7
    call _f

    ; Вывод результата
    mov eax, result
    call sprint
    mov eax, [res]
    call iprintLF
    call quit

_g:
    mov ebx, 3
    mul ebx
    sub eax, 1
    ret

_f:
    mov ebx, 2
    mul ebx
    add eax, 7
    mov [res], eax
    ret
```

Результат выполнения:

```bash
@uakhafiz → ~/work/arch-pc/lab09$ ./lab09-1
Введите x: 2
f(g(x))=2*(3x-1)+7=17
```

Верификация вычислений:

· g(2) = 3×2 - 1 = 5
· f(5) = 2×5 + 7 = 17 ✓

4. Выполнение самостоятельной работы

4.1 Поиск и исправление ошибки в программе

Исходная программа: Вычисление выражения (3 + 2) × 4 + 5

Обнаруженная проблема: Программа выдавала неверный результат (10 вместо 25)

```bash
@uakhafiz → ~/work/arch-pc/lab09$ ./lab09-4
Результат: 10
```

Анализ в GDB:

```asm
mov ebx, 3          ; ebx = 3
mov eax, 2          ; eax = 2
add ebx, eax        ; ebx = 5 (3+2)
mov ecx, 4          ; ecx = 4
mul ecx             ; eax = 2 × 4 = 8 (ОШИБКА!)
add ebx, 5          ; ebx = 5 + 5 = 10
```

Выявленная ошибка: Инструкция mul ecx умножает регистр eax на ecx, но в eax находится значение 2, а не результат сложения (5).

4.2 Исправленная версия программы

Исправленный код:

```asm
; lab09-4.asm (исправленный)
; Программа вычисления (3+2)*4+5

%include 'in_out.asm'

SECTION .data
    div: DB 'Результат: ',0

SECTION .text
    GLOBAL _start

_start:
    ; --- Вычисление выражения (3+2)*4+5
    mov eax, 3
    mov ebx, 2
    add eax, ebx      ; eax = 5 (3+2)
    mov ecx, 4
    mul ecx           ; eax = 5 × 4 = 20
    add eax, 5        ; eax = 20 + 5 = 25
    mov edi, eax      ; Сохранение результата

    ; --- Вывод результата на экран
    mov eax, div
    call sprint
    mov eax, edi
    call iprintLF
    call quit
```

Результат выполнения:

```bash
@uakhafiz → ~/work/arch-pc/lab09$ ./lab09-4
Результат: 25
```

4.3 Верификация исправления

Пошаговая проверка:

1. (3 + 2) = 5
2. 5 × 4 = 20
3. 20 + 5 = 25

Вывод: Программа работает корректно, вычисления верифицированы.
