## 2. Теоретическое введение

### 2.1 Организация стека в процессоре

Стек — это фундаментальная структура данных, организованная по принципу **LIFO** (Last In - First Out). В архитектуре x86 стек реализован на аппаратном уровне и играет важную роль в выполнении программ.

**Ключевые характеристики стека:**
- Организован как область оперативной памяти
- Управляется через регистры `ESP` (указатель стека) и `EBP` (базовый указатель)
- Растет в сторону младших адресов памяти
- Используется для хранения адресов возврата, передачи параметров и локальных переменных

**Базовые операции:**
| Инструкция | Описание |
|------------|----------|
| `push` | Добавление элемента в вершину стека |
| `pop` | Извлечение элемента из вершины стека |

### 2.2 Инструкции организации циклов

Инструкция `loop` предоставляет эффективный механизм для организации циклов с фиксированным количеством итераций.

**Алгоритм работы:**
```asm
mov ecx, N          ; Инициализация счетчика
label:
    ; Тело цикла
    loop label       ; ECX = ECX - 1; if ECX ≠ 0 goto label
```

Особенности:

· Регистр ECX используется как счетчик итераций
· Каждая инструкция loop уменьшает ECX на 1
· Цикл завершается при ECX = 0

2.3 Обработка аргументов командной строки

При запуске программы в NASM аргументы командной строки загружаются в стек в определенном порядке:

Порядок элементов в стеке:

1. Количество аргументов (argc)
2. Имя программы (argv[0])
3. Аргумент 1 (argv[1])
4. Аргумент 2 (argv[2])
5. ... и так далее

Для корректной обработки необходимо извлечь количество аргументов и затем в цикле обработать каждый аргумент.

3.3 Анализ проблем при работе с циклом

При модификации программы с добавлением инструкции sub ecx,1 внутри цикла возникла критическая проблема.

Проблемный код:

```asm
label:
    sub ecx, 1        ; Некорректное изменение ECX
    mov [N], ecx
    mov eax, [N]
    call iprintLF
    loop label
```

Анализ проблемы: Инструкция loop самостоятельно уменьшает ECX на 1, поэтому дополнительное уменьшение приводит к непредсказуемому поведению и потенциально бесконечному циклу.

3.4 Решение проблемы с использованием стека

Для корректной работы цикла при необходимости изменения ECX в теле цикла применяется механизм сохранения/восстановления значения через стек.

Исправленный код:

```asm
label:
    push ecx           ; Сохранение ECX в стек
    sub ecx, 1         ; Работа с копией значения
    mov [N], ecx
    mov eax, [N]
    call iprintLF
    pop ecx            ; Восстановление оригинального ECX
    loop label
```

Результат: Программа работает корректно, выводя ровно N итераций. Демонстрируется важность сохранения состояния регистров в сложных циклических конструкциях.

3.5 Программа lab8-2.asm - Обработка аргументов командной строки

_end:
    call quit

3.6 Программа lab8-3.asm - Математические операции над аргументами

Задача: Создание программы для вычисления произведения аргументов командной строки.

Исходный код:

```asm
; lab8-3.asm
; Программа для вычисления произведения аргументов командной строки

%include 'in_out.asm'

SECTION .data
    msg: DB "Результат: ",0

SECTION .text
    GLOBAL _start

_start:
    pop ecx            ; argc
    pop edx            ; argv[0] (имя программы)
    sub ecx, 1         ; Количество аргументов (без имени программы)
    mov esi, 1         ; Инициализация аккумулятора для умножения

next:
    cmp ecx, 0
    jz _end            ; Если аргументов нет, завершаем
    pop eax            ; Извлечение аргумента
    call atoi          ; Преобразование строки в число
    imul esi, eax      ; Умножение аккумулятора на текущее число
    loop next

_end:
    mov eax, msg
    call sprint
    mov eax, esi
    call iprintLF
    call quit
```

Особенности реализации:

· Использование регистра ESI как аккумулятора для накопления результата
· Применение инструкции imul для знакового умножения
· Преобразование строковых аргументов в числовой формат с помощью atoi

4. Выполнение самостоятельной работы

4.1 Постановка задачи

Разработать программу, которая вычисляет сумму значений функции f(x) = 2x + 15 для заданных значений x, передаваемых как аргументы командной строки.

Математическая формулировка:
Для x₁, x₂, ..., xₙ вычислить:

```
S = f(x₁) + f(x₂) + ... + f(xₙ) = Σ(2xᵢ + 15)
```

4.2 Разработка алгоритма

Алгоритм программы:

1. Извлечь количество аргументов из стека
2. Инициализировать аккумулятор суммы нулем
3. Для каждого аргумента:
   · Преобразовать строку в число
   · Вычислить f(x) = 2x + 15
   · Добавить результат к аккумулятору
4. Вывести полученную сумму

4.3 Реализация программы main.asm

```asm
; main.asm
; Программа для вычисления суммы f(x) = 2x + 15 для аргументов командной строки

%include 'in_out.asm'

SECTION .data
    msg_func: DB 'Функция: f(x)=2x+15',10,0
    msg_result: DB 'Результат: ',0

SECTION .text
    GLOBAL _start

_start:
    ; Вывод информации о функции
    mov eax, msg_func
    call sprint

    ; Получение аргументов командной строки
    pop ecx            ; argc (количество аргументов)
    pop edx            ; argv[0] (имя программы)
    sub ecx, 1         ; Уменьшаем счетчик на 1 (пропускаем имя программы)

    mov esi, 0         ; Инициализация аккумулятора суммы (ESI = 0)

next_arg:
    cmp ecx, 0
    jz print_result    ; Если аргументов нет, переходим к выводу результата

    pop eax            ; Извлечение аргумента
    call atoi          ; Преобразование строки в число

    ; Вычисление f(x) = 2x + 15
    mov ebx, 2
    mul ebx            ; EAX = 2 * x
    add eax, 15        ; EAX = 2x + 15

    ; Добавление к сумме
    add esi, eax       ; ESI = ESI + f(x)

    loop next_arg      ; Переход к следующему аргументу

print_result:
    ; Вывод результата
    mov eax, msg_result
    call sprint
    mov eax, esi
    call iprintLF

    call quit
```
